# 고객 목록 관리 페이지 기획 및 설계서

## 1. 개요
본 문서는 사용자(고객) 목록을 관리하기 위한 웹 페이지의 UI/UX, 주요 기능 및 데이터 구조를 정의한 명세서입니다.

## 2. 주요 기능 (Features)

### 2.1. 고객 목록 조회 (Read)
* **테이블 형태의 데이터 출력**: 이름, 이메일, 연락처, 가입일, 상태 등의 정보를 테이블 형태로 표시합니다.
* **페이징 (Pagination)**: 데이터가 많을 경우 한 페이지당 10~20개의 항목만 보이도록 페이징 처리를 합니다.
* **정렬 (Sorting)**: 각 열(가입일, 이름 등)의 헤더를 클릭하여 오름차순/내림차순으로 정렬할 수 있습니다.

### 2.2. 검색 및 필터링 (Search & Filter)
* **통합 검색**: 이름, 이메일, 연락처를 기반으로 특정 고객을 검색할 수 있는 검색 바를 제공합니다.
* **상태 필터링**: 활성/비활성 사용자, 등급별 필터링 기능을 제공합니다.

### 2.3. 고객 추가 (Create)
* 신규 고객을 등록할 수 있는 '고객 추가' 버튼 및 모달(Modal) 혹은 별도 폼 페이지를 제공합니다.
* 유효성 검사(Validation): 이메일 형식, 필수 입력 값 누락 등을 검사합니다.

### 2.4. 고객 정보 수정 (Update)
* 목록에서 개별 고객을 클릭하거나 '수정' 버튼을 눌러 고객의 상세 정보를 변경할 수 있습니다.

### 2.5. 고객 삭제 (Delete)
* 개별 삭제 및 다중 선택 삭제(체크박스 활용) 기능을 제공합니다.
* 삭제 시 실수 방지를 위해 확인 창(Confirm)을 띄웁니다.

## 3. 화면 UI 레이아웃 설계

* **상단 (Header/Top Bar)**
  * 페이지 타이틀 ("고객 관리")
  * 검색 바 (Search Input) 및 필터링 옵션 셀렉트 박스
  * "고객 추가" 버튼 (Primary 색상 적용)

* **본문 (Main Content)**
  * **데이터 테이블**:
    | 체크박스 | ID | 이름 | 이메일 | 연락처 | 가입경로 | 가입일 | 상태 | 액션(수정/삭제) |
  * **페이징 컨트롤러**: 본문 하단에 [1] [2] [3]... 및 이전/다음 버튼 배치

## 4. 데이터베이스 구조 (예시)

고객(Customers) 테이블 예시 구조:
* `id` (UUID 또는 Auto-increment INT, Primary Key)
* `name` (VARCHAR, 필수)
* `email` (VARCHAR, 필수, 고유값)
* `phone` (VARCHAR)
* `signup_path` (VARCHAR, 가입경로 예: 'GOOGLE', 'NAVER', 'EMAIL')
* `status` (VARCHAR, 예: 'ACTIVE', 'INACTIVE')
* `created_at` (TIMESTAMP, 기본값 NOW())
